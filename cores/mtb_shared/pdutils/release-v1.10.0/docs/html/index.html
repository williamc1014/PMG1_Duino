PDUtils
PDUtils middleware library


The PDUtils middleware implements the software timer and utility functions required by EZ-PD(TM) PMG1/CCGx/WLC devices.
Features:

  1. Provides APIs for software timer management
  2. Provides APIs for performing utility functions such as Checksum, CRC check, and event management.

General description

Include cy_pdutils.h and cy_pdutils_sw_timer.h to get access to all functions and other declarations in this library. See the Quick start guide to start using PDUtils.

See the Supported software and tools section for compatibility information.

See the Changelog section for change history.

The PDUtils middleware implements the software timer and utility functions that are needed for the EZ-PD(TM) PMG1/CCGx/WLC solutions.

The software timer module provides a WDT/SYS_TICK-based soft timer implementation. This allows different modules to create millisecond-level timers without additional hardware support. The pdutils middleware defines ranges of timer IDs for different usage as shown in the following table.

Section name Section index Timer ID range
Device-specific 0 0x0000 - 0x00FF
PD and Type-C Stack (Port 0) 1 0x0100 - 0x01FF
PD and Type-C Stack (Port 1) 2 0x0200 - 0x02FF
Base application stack (Port 0) 3 0x0300 - 0x03FF
Base application stack (Port 1) 4 0x0400 - 0x04FF
Alternate mode stack (Port 0) 5 0x0500 - 0x05FF
Alternate mode stack (Port 1) 6 0x0600 - 0x06FF
Wireless charging 7 0x0700 - 0x07FF
Solution layer 8 0x0800 - 0x08FF
Reserved for future usage 9 - 15 0x0900 - 0x0FFF
User space 16 - 255 0x1000 - 0x1FFF

The actual timer ID allocation and management within the available range is done by the respective assets/modules. The device-specific timer IDs are defined and managed by the PDUtils middleware.

Quick start guide

These steps describe the simplest way of enabling the PDUtils software timers in the application.

  1. Includes the cy_pdutils.h and cy_pdutils_sw_timer.h to get access to all functions and other declarations of this middleware.
    #include "cy_pdutils.h"
    #include "cy_pdutils_sw_timer.h"
  2. Defines the data structures required by the PDUtils software timer.
    cy_stc_pdutils_sw_timer_t glSwTimerContext;
    cy_stc_pdutils_timer_config_t glSwTimerConfig;
    void *callerContext;
  3. Initializes the PDUtils software timer module after the start. If the timer type is watchdog(WDT) timer, then register the interrupt handler for the watchdog timer. The watchdog timer is used to implement the software timers.
    const cy_stc_sysint_t wdt_interrupt_config =
    {
    .intrSrc = (IRQn_Type)srss_interrupt_wdt_IRQn,
    .intrPriority = 0U,
    };
    Cy_SysInt_Init(&wdt_interrupt_config, &wdt_interrupt_handler);
    NVIC_EnableIRQ(wdt_interrupt_config.intrSrc);
    Shows a reference wdt_interrupt_handler implementation.
    void wdt_interrupt_handler(void)
    {
    /* Clear WDT pending interrupt */
    Cy_WDT_ClearInterrupt();
    #if (CY_PDUTILS_TIMER_TICKLESS_ENABLE == 0)
    /* Load the timer match register. */
    Cy_WDT_SetMatch((Cy_WDT_GetCount() + glSwTimerContext.multiplier));
    #endif /* (CY_PDUTILS_TIMER_TICKLESS_ENABLE == 0) */
    /* Invoke the timer handler. */
    Cy_PdUtils_SwTimer_InterruptHandler (&(glSwTimerContext));
    }
    Invokes the Cy_PDUtils_SwTimer_Init by passing required parameters.
    glSwTimerConfig.sys_clk_freq = Cy_SysClk_ClkSysGetFrequency();
    glSwTimerConfig.hw_timer_ctx = NULL;
    /* Initialize the soft timer module. */
    Cy_PdUtils_SwTimer_Init(&glSwTimerContext, &glSwTimerConfig);
    /* Enable global interrupts */
    __enable_irq();

  4. Starting timers
    Starts a specific software timer by passing the timer ID and desired timer period (ms). All software timers are one-shot timers that will run until the specified period has elapsed. The timer expiration callback will be called at the end of the period if registered.

    Defines the timer ID and timer period.

    /* User Timer ID */
    #define USER_TIMER_ID (0x1000)
    /* User Timer Period in ms */
    #define USER_TIMER_PERIOD_MS (750u)

    References the callback for an expiry notification handling.

    void user_timer_cb (
    cy_timer_id_t id,
    void *callbackContext)
    {
    /* Timer Expiry handling */
    (void)id;
    (void)callbackContext;
    }

    Starts a timer by passing the caller context, timer ID, desired period in ms, and a callback for an expiry notification.

    retVal = Cy_PdUtils_SwTimer_Start(&glSwTimerContext, callerContext, USER_TIMER_ID, USER_TIMER_PERIOD_MS, user_timer_cb);
    if (retVal == true)
    {
    /* Timer start successful. */
    }

    Uses when a callback for an expiry notification is not required.

    retVal = Cy_PdUtils_SwTimer_StartWocb(&glSwTimerContext, USER_TIMER_ID, USER_TIMER_PERIOD_MS);
    if (retVal == true)
    {
    /* Timer start successful. */
    }
  5. Checks the timer running status.
    retVal = Cy_PdUtils_SwTimer_IsRunning(&glSwTimerContext, USER_TIMER_ID);
    if (retVal == true)
    {
    /* USER_TIMER_ID is running. */
    }
  6. Stopping timers
    Stops a timer that is currently running:
    Cy_PdUtils_SwTimer_Stop(&glSwTimerContext, USER_TIMER_ID);
    Stops all active software timers:
    Cy_PdUtils_SwTimer_StopAll(&glSwTimerContext);
    Stops software timers with timer IDs in the specified range:
    Cy_PdUtils_SwTimer_StopRange(&glSwTimerContext, USER_TIMER_ID_START, USER_TIMER_ID_END);


In addition to the software timer functionality, the PDUtils middleware provides APIs for event management and other utility functions. The following steps describe the usage of these functions in the application.

  1. Event management functions
    An event group facilitates the management of a set of related tasks or status bits. The following APIs can be used to set, get, and clear events for a event group. Event groups are DWORD(uint32_t) variables that hold a set of request bits that can be set/cleared independently.

    Event group variable.

    uint32_t eventMap = 0;

    Setting a bit in an event group variable.

    Cy_PdUtils_EventGroup_SetEvent(&eventMap, 4);

    Setting one or more values in an event group variable.

    Cy_PdUtils_EventGroup_SetEventsByVal(&eventMap, 2);

    Get the position of the event detected that needs to be processed.

    evt = Cy_PdUtils_EventGroup_GetEvent(&eventMap, true);

    Clears a bit in an event group variable.

    Cy_PdUtils_EventGroup_ClearEvent(&eventMap, 1);

    Clears one or more values in an event group variable.

    Cy_PdUtils_EventGroup_ClearEventsByVal(&eventMap, 16);
  2. Other Utility functions
    Computes the checksum (2's complement of the binary sum of all bytes) for the specified byte array.
    byteChecksum = Cy_PdUtils_MemCalculateByteChecksum(&byteData[0], 32);

    Computes the checksum (2's complement of the binary sum of all words) for the specified WORD array.
    wordChecksum = Cy_PdUtils_MemCalculateWordChecksum(&wordData[0], 32);

    Computes the checksum (2's complement of the binary sum of all words) for the specified DWORD array.
    dwordChecksum = Cy_PdUtils_MemCalculateDwordChecksum(&dwordData[0], 32);

    Implements the CRC-16 with polynomial x^16 + x^15 + x^2 + 1 (0xA001).
    newCrc = Cy_PdUtils_Crc16(crc, 0x07);

    Returns the quotient rounded up to the nearest integer for the given two unsigned integers.
    quotient = Cy_PdUtils_DivRoundUp(4, 2);

Configuration Considerations

This section consists of instructions on how to configure and use the PDUtils middleware in a design.

  1. After including the PDUtils files, select the timer type by setting:
    • CY_PDUTILS_TIMER_TYPE to CY_PDUTILS_TIMER_TYPE_SYSTICK for using SYSTICK timer based software timers.
    • CY_PDUTILS_TIMER_TYPE to CY_PDUTILS_TIMER_TYPE_WDT for using Watchdog timer based software timers.
    • CY_PDUTILS_TIMER_TYPE to CY_PDUTILS_TIMER_TYPE_PD_ILO for using LF timer based software timers.
  2. Set the macro CY_PDUTILS_TIMER_CALIB_SAMPLE_TIME_1MS to '1' to enable timer calibration sample time of 1 ms. By default, this macro is set to '0' for a sample time of 10 ms.


See the Limitations and restrictions section for the existing restrictions.

Limitations and restrictions

  1. The timer source used by the software timer should not be used elsewhere in the application.
